The "Key-Value List" (KVL)

An enhanced, one level JSON data editor.

Simple (empty) list
Customized list with starting data 
<!-- Simple Key-Value List -->
<div class="mdl-kvl mdl-js-kvl">
    <textarea name="custom1" id="custom1" class="mdl-kvl__input"></textarea>
    <label class="mdl-kvl__label" for="custom1">My Custom list</label>
</div>
<!-- Customized Key-Value List -->
<div class="mdl-kvl mdl-js-kvl">
    <textarea name="custom2" id="custom2" class="mdl-kvl__input" data-key-placeholder="Title" data-value-placeholder="Description">{"Weather":"Sunny","Mood":"Great"}</textarea>
    <label class="mdl-kvl__label" for="custom2">What's up?</label>
</div>

Introduction

The Key-Value List (KVL) component is an enhanced, one level JSON data editor, build on the standard HTML <textarea> element. After the component is activated, the <textarea> element is not visible any more, instead a new add button appears which adds new key-value option rows to the component.

While the user simply enters custom (or provided) keys and value pairs on the frontend, the component transform the given data into JSON as the <textarea> element's value.

To include a KVL component:

You can use this component as any other official MDL component, except this component needs individual JavaScript and CSS file to be included since this component is not part of the official MDL library.

 1. Code a <div> element to hold the text field.

<div>...</div>

 2. Inside the <div>, code a <textarea> element with an id and name attribute of your choice.

<div>
    <textarea name="custom" id="custom"></textarea>
</div>

 3. Also inside the <div>, after the <textarea> element, code a <label> element with a for attribute whose value matches the <textarea> element's id value, and type a short string to be used as the field's title / label.

<div>
    <textarea name="custom" id="custom"></textarea>
    <label for="custom">My Custom List</label>
</div>

 4. Add one or more classes, separated by spaces, to the <div> container, the <textarea> element, and the <label> element using the class attribute.

<div class="mdl-kvl mdl-js-kvl">
    <textarea name="custom" id="custom" class="mdl-kvl__input"></textarea>
    <label class="mdl-kvl__label" for="custom">My Custom List</label>
</div>

The KVL component is ready for use.

To fine-tune a KVL component:

The KVL component is more then just a blank container, you can specify some really useful behaviour as well.

 1. Add data-key-placeholder and/or data-value-placeholder attributes to the <textarea> element to define field placeholders.

Note: Because of the limited size of the fields you have to use short keywords.

<div class="mdl-kvl mdl-js-kvl">
    <textarea name="custom" id="custom" class="mdl-kvl__input" data-key-placeholder="Title" data-value-placeholder="Description"></textarea>
    <label class="mdl-kvl__label" for="custom">My Custom List</label>
</div>

 2. There's a possibility the add a prepared data list for the keys in some special cases:

  • Phone Numbers
  • Social Networks
  • Instant Messengers

Note: These lists are not fully covers the possibilities but give a good start by providing the most popular cases

To enable this feature you only need to add a special class attribute on the <div> element:

  • mdl-kvl-phones for phone numbers
  • mdl-kvl-socialnetworks for the social networks
  • mdl-kvl-instantmessengers for the instant messengers
<div class="mdl-kvl mdl-js-kvl mdl-kvl-phones">
    <textarea name="phone" id="phone" class="mdl-kvl__input"></textarea>
    <label class="mdl-kvl__label" for="phone">Contact phones</label>
</div>

 3. You can also set what kind of data you need for the values. For this just add a data-type attribute to the <textarea> element, with one of the supported types:

  • text for any type of text
  • email for email addresses
  • url for internet addresses
  • tel for phone numbers.
<div class="mdl-kvl mdl-js-kvl mdl-kvl-phones">
    <textarea name="phone" id="phone" data-type="tel" class="mdl-kvl__input"></textarea>
    <label class="mdl-kvl__label" for="phone">Contact phones</label>
</div>

As a bonus: also add the data-pattern attribute to activate the input validation. If the attribute value is empty, then the script will provide a default one according to the data-type.

<div class="mdl-kvl mdl-js-kvl mdl-kvl-phones">
    <textarea name="phone" id="phone" data-type="tel" data-pattern="^\s*\(?(020[7,8]{1}\)?[ ]?[1-9]{1}[0-9{2}[ ]?[0-9]{4})|(0[1-8]{1}[0-9]{3}\)?[ ]?[1-9]{1}[0-9]{2}[ ]?[0-9]{3})\s*$" class="mdl-kvl__input"></textarea>
    <label class="mdl-kvl__label" for="phone">UK Phones Numbers</label>
</div>

Configuration options

The CSS classes apply various predefined visual enhancements to the Key-Value List. The table below lists the available classes and their effects.

MDL Class Effect Remarks
mdl-kvl Defines container as an MDL component Required on "outer" div element
mdl-js-kvl Assigns basic MDL behavior to input Required on "outer" div element
mdl-kvl-instantmessengers Activates the predefined "IM" autocomplete list Optional on "outer" div element
mdl-kvl-socialnetworks Activates the predefined "Social Networks" autocomplete list Optional on "outer" div element
mdl-kvl-phones Activates the predefined "Phone group" autocomplete list Optional on "outer" div element
mdl-kvl__label Defines element as textarea label Required on label element for textarea elements
mdl-kvl__input Defines element as datacontainer input Required on textarea element

File Input

An enhanced file uploader

File input with floating label 
<!-- File input -->
<div class="mdl-file mdl-js-file mdl-file--floating-label">
    <input type="file" name="avatar" id="avatar">
    <label class="mdl-file__label" for="avatar">Avatar</label>
</div>

Note: To unset/reset the file input only, you have to press the "attach" button again and on the popup file selection window, press the Cancel button.

Introduction

The File Input component is an enhanced file input field, build on the standard <input type="file"> element. After the component is activated, the <input type="file"> element is not visible any more, instead a new attach <button> and an MDL <input> element appear which give the eye-candy.

To include a File Input component:

You can use this component as any other official MDL component, except this component needs individual JavaScript and CSS file to be included since this component is not part of the official MDL library.

 1. Code a <div> element to hold the text field.

<div>...</div>

 2. Inside the <div>, code a <input> element with an id and name attribute of your choice. You must set the type attribute to file.

<div>
    <input type="file" name="avatar" id="avatar">
</div>

 3. Also inside the <div>, after the <input> element, code a <label> element with a for attribute whose value matches the <input> element's id value, and type a short string to be used as the field's label.

<div>
    <input type="file" name="avatar" id="avatar">
    <label for="avatar">Avatar</label>
</div>

 4. Add one or more classes, separated by spaces, to the <div> container and the <label> element using the class attribute.

<div class="mdl-file mdl-js-file mdl-file--floating-label">
    <input type="file" name="avatar" id="avatar">
    <label class="mdl-file__label" for="avatar">Avatar</label>
</div>

The File Input component is ready for use.

Examples

A file input with custom text when multiple files are selected.

<div class="mdl-file mdl-js-file mdl-file--floating-label" data-placeholder-multiple="files had been added to the list">
    <input type="file" name="avatar" id="avatar" multiple="multiple">
    <label class="mdl-file__label" for="avatar">Avatar</label>
</div>

Note: The data-placeholder-multiple attribute requires the multiple attribute to present.

Note: The content of the data-placeholder-multiple attribute will be concatenated to the number of the selected files, like: "3 files had been added to the list"

Configuration options

The CSS classes apply various predefined visual enhancements to the file input. The table below lists the available classes and their effects.

MDL class Effect Remarks
mdl-file Defines container as an MDL component Required on "outer" div element
mdl-js-file Assigns basic MDL behavior to input Required on "outer" div element
mdl-file--floating-label Applies floating label effect Optional on "outer" div element
mdl-file__label Defines element as textarea label Required on label element for textarea elements

Avatar

An enhanced avatar file uploader

The most basic avatar uploader setup. 
<!-- Avatar -->
<div class="mdl-avatar mdl-js-avatar mdl-avatar--floating-label">
    <input class="mdl-avatar__input" type="text" name="profile_image" id="profile_image" value="img/avatars/default1.png">
    <label class="mdl-avatar__label" for="profile_image">Profile Image</label>
</div>

Note: You have to set enctype="multipart/form-data" as the <form> element's attribute to be able to upload any file.

Introduction

The Avatar component is an enhanced file uploader, which combines many MDL components, like `List`, `Fab buttons`, `Input fields` and also another Component Extension the `File input`.

To include an Avatar component:

You can use this component as any other official MDL component, except this component needs individual JavaScript and CSS file to be included since this component is not part of the official MDL library.

 1. Code a <div> element to hold the text field.

<div>...</div>

 2. Inside the <div>, code a <input type="text"> element with an id and name attribute of your choice. You also have to set the value attribute to the initial location of the avatar (e.g.: a default or blank image)

<div>
    <input type="text" name="profile_image" id="profile_image" value="img/avatars/default1.png">
</div>

 3. Also inside the <div>, after the <input> element, code a <label> element with a for attribute whose value matches the <input> element's id value, and type a short string to be used as the field's label.

<div>
    <input type="text" name="profile_image" id="profile_image" value="img/avatars/default1.png">
    <label for="profile_image">Profile Image</label>
</div>

 4. Add one or more classes, separated by spaces, to the <div> container and the <label> element using the class attribute.

<div class="mdl-avatar mdl-js-avatar mdl-avatar--floating-label">
    <input class="mdl-avatar__input" type="text" name="profile_image" id="profile_image" value="img/avatars/default1.png">
    <label class="mdl-avatar__label" for="profile_image">Profile Image</label>
</div>

The File Input component is ready for use.

To fine-tune an Avatar component:

There are some tricks in the pocket to control the component

 1. Add data-gallery-src and data-gallery-list attributes to the <input> element to enable a gallery preset for the component. The value of the data-gallery-list attribute MUST be a comma-separated list. The value of the value attribute will be added to the list automatically, so you can leave it out from the list.

Note: In the gallery list you can specify the images with absolute path (start with '/') too.

<div class="mdl-avatar mdl-js-avatar mdl-avatar--floating-label">
    <input class="mdl-avatar__input" type="text" name="profile_image" id="profile_image" value="/img/avatars/default1.png" data-gallery-src="/img/avatars/" data-gallery-list="default2.png,default3.png,default4.png">
    <label class="mdl-avatar__label" for="profile_image">Profile Image</label>
</div>

 2. Add data-i18n-* attributes to the <input> element to set translations or custom labels for the four options: 'Gallery', 'GR Avatar', 'URL', 'Upload File'.

Note: The label size is limited, so the too long texts will be cut half.

<div class="mdl-avatar mdl-js-avatar mdl-avatar--floating-label">
    <input class="mdl-avatar__input" type="text" name="profile_image" id="profile_image" value="/img/avatars/default1.png" data-i18n-gallery="Default" data-i18n-gravatar="GRavatar email" data-i18n-url="Internet link" data-i18n-upload="From local machine">
    <label class="mdl-avatar__label" for="profile_image">Profile Image</label>
</div>

 3. Using the examples above the component will POST three data by default:


array (size=3)
    'profile_image' => string '/img/avatar/default1.png' (length=24)
    'profile_image-type' => string 'gallery' (length=7)
    'profile_image-file' =>
        array (size=5)
            'name' => string '' (length=0)
            'type' => string '' (length=0)
            'tmp_name' => string '' (length=0)
            'error' => int 4
            'size' => int 0

... or when uploading a file:


array (size=3)
    'profile_image' => string 'custom_profile_image.png' (length=24)
    'profile_image-type' => string 'upload' (length=6)
    'profile_image-file' =>
        array (size=5)
            'name' => string 'custom_profile_image.png' (length=24)
            'type' => string 'image/png' (length=10)
            'tmp_name' => string '/tmp/phpxu2KUo' (length=14)
            'error' => int 0
            'size' => int 95079

Note: Depending on the script language or application on the server-side, the uploaded file may presented in a different way.

If your application - for validation purposes - requires the form to render all the elements that will submit data, you may specify the type and file elements and you can also give custom field names to them:

<div class="mdl-avatar mdl-js-avatar mdl-avatar--floating-label">
    <input class="mdl-avatar__input" type="text" name="profile_image" id="profile_image" value="img/avatars/default1.png">
    <input type="hidden" class="mdl-avatar__type" name="profile_type" value="">
    <input type="file" name="profile_file">
    <label class="mdl-avatar__label" for="profile_image">Profile Image</label>
</div>

Note: There are some restrictions you must keep:

  • * Do not use any custom containers for the <input type="file"> and for the avatar type input.
  • * The input element for the avatar type must use the type="hidden" attribute.
  • * The input element for the avatar type must use the class="mdl-avatar__type" attribute.

Examples

A full configuration component 
<div class="mdl-avatar mdl-js-avatar mdl-avatar--floating-label">
    <input class="mdl-avatar__input" type="text" name="profile[image]" id="profile[image]" value="img/avatars/default1.png" data-gallery-src="/img/avatars/" data-gallery-list="default2.png,default3.png,default4.png"  data-i18n-gallery="Default" data-i18n-gravatar="GRavatar email" data-i18n-url="Internet link" data-i18n-upload="From local machine">
    <input type="hidden" class="mdl-avatar__type" name="profile[type]" value="">
    <input type="file" name="profile[file]">
    <label class="mdl-avatar__label" for="profile[image]">Profile Image</label>
</div>
